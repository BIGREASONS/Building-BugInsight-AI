## 1. Problem Statement

### What Software Teams Struggle With Today

Modern engineering teams are routinely overwhelmed by thousands of security alerts generated by static analysis (SAST) and dependency scanning tools. While finding vulnerabilities has been automated, the response phase remains fundamentally broken:

* **Triage Churn:** Engineers spend hours manually sifting through false positives and trying to determine if a vulnerability is reachable in production.
* **Context Switching:** Tracing an execution path through an unfamiliar or legacy codebase to find the root cause requires massive cognitive effort.
* **Remediation Bottlenecks:** Manually writing a safe fix, validating the syntax to ensure it doesn't break logic, and opening pull requests consumes days of valuable development cycles, leaving security windows open for far too long.

### Why Existing Tools Are Insufficient

* **Traditional SAST Linters:** They find pattern matches but lack deep semantic awareness of execution paths, runtime impact, or the business context necessary to fix the bug safely.
* **Standard AI Coding Assistants:** Autocomplete windows and reactive chat interfaces are localized to a developer's active IDE file. They do not run continuously at the repository level, cannot orchestrate multi-step validation checks, and often leak sensitive proprietary vulnerabilities to public cloud endpoints.

### Why AI Agents Are Needed

Shifting from *assistance* to true *autonomy* requires shifting human labor from operational execution to supervisory validation. A single monolithic LLM prompt fails due to token context window degradation and reasoning loss over large repositories. **BugInsight Swarm** solves this by deploying a distributed squad of single-responsibility AI agents that share a unified state machine—collaborating to discover, analyze, validate, and patch code defects without human intervention.

## 2. Project Vision & Core Platform

**BugInsight Swarm** is an autonomous, AI-native security engineering platform that analyzes repositories and handles the entire lifecycle of a software vulnerability. Given a repository URL or an upstream alert, the system activates: cloning the codebase, mapping its structural layout, assessing real-world severity via a CodeBERT-based classification layer, isolating the root cause, synthesizing a verified security patch, and automatically delivering a production-ready GitHub Pull Request.

---

## 3. System Architecture & MVP Design

BugInsight Swarm avoids the complexity of heavy enterprise cloud infrastructure in favor of a lean, high-visibility, **zero-data-leakage** multi-agent architecture built using **LangGraph** and local inference.

### The Core Five MVP Agents

* **Repository Analysis Agent:** Extracts code layout topology, maps component dependencies, and fetches relevant code snippets using Tree-sitter and ChromaDB.
* **CodeBERT Triage Agent:** Performs deep semantic risk evaluation on targeted code paths to output concrete engineering severity using a CodeBERT-based semantic vulnerability classification layer.
* **Root Cause Agent:** Analyzes surrounding code context, repository structure, scanner findings, and semantic signals to isolate the root cause using a local reasoning LLM (`llama3.1:8b` via Ollama).
* **Patch Generation Agent:** Synthesizes code remedies that neutralize security flaws without introducing breaking syntax changes using a specialized coding LLM (`qwen2.5-coder:7b` via Ollama).
* **Automated Pull Request Agent:** Forks or branches the target repository, writes the change directly to the file system, verifies the patch via AST checks and security rescans, and automatically deploys a live upstream Pull Request using PyGithub.

## 4. Tech Stack & Engineering Innovations

### The Zero-Leakage Advantage

* **Orchestration & Workflow:** FastAPI backend paired with LangGraph for deterministic state-machine management between agents.
* **Local Inference & Vector Search:** ChromaDB for repository codebase vector retrieval, combined entirely with local LLMs (`qwen2.5-coder:7b`, `llama3.1:8b`) running via **Ollama**. This guarantees that sensitive, vulnerable proprietary code never leaves the organization's hardware.
* **Specialized ML Layer:** CodeBERT-based semantic vulnerability classification handling deep-semantic security routing.
* **Observability:** Custom Next.js real-time telemetry dashboard utilizing Server-Sent Events (SSE) to display live agent execution steps.

### Reliability Innovation: AST-Based Surgical Window Splicing (SWS)

During development, we discovered that small local language models frequently failed when asked to regenerate entire source files. This led to truncated outputs, dropped imports, and syntax regressions.

To solve this, BugInsight Swarm uses **AST-Based Surgical Window Splicing (SWS)**. Instead of regenerating full files, the system extracts *only* the vulnerable function or localized code region, generates a targeted remediation, and deterministically splices the fix back into the original source file. This dramatically reduces hallucinations, preserves surrounding code byte-for-byte, and enables highly reliable patch generation using fully local models running through Ollama.

## 5. Live Scenario Proof: Neutralizing Unsafe Deserialization

During our live validation run, BugInsight Swarm was pointed directly at a repository containing an unvalidated object deserialization flaw (a critical security bottleneck that traditional linters fail to contextualize).

* **Triage & Classification:** The platform's integrated CodeBERT layer flagged the specific untrusted input stream, classifying it as a **Critical** risk based on semantic code analysis and scanner findings.
* **Remediation & Double-Loop Validation:** The Patch Agent automatically replaced the unsafe deserialization API with a safer JSON-based parsing approach (`json.loads`) and verified removal of the original vulnerability pattern. The framework validated the patch through rigorous AST syntax checks and ran a clean post-remediation security scan.
* **Autonomous Delivery:** The system created a dedicated hotfix branch, pushed the rectified code upstream, and opened a live GitHub Pull Request complete with structural validation checklists. During validation, the platform successfully generated a live GitHub Pull Request containing the proposed remediation, demonstrating end-to-end repository analysis, patch generation, validation, and developer handoff.

BugInsight Swarm cuts the timeframe between vulnerability discovery and source code remediation from days down to seconds, ensuring developers stay firmly in control as final code reviewers.
